## Project Overview

MAEP (Minimum Viable Agent-to-Agent Economic Protocol) — a blockchain-based protocol for autonomous AI agents to discover each other, delegate tasks, settle payments trustlessly, and verify results on-chain. Dual-language: Solidity/JavaScript (smart contracts) + Python (agent SDK and experiments).

## Build & Run Commands

### Install Dependencies
```bash
npm install
pip install -r requirements.txt
```

### Smart Contracts (Hardhat)
```bash
npx hardhat compile                                    # Compile Solidity
npx hardhat test                                       # Run all contract tests
npx hardhat test test/PaymentChannel.test.js           # Single test file
node scripts/export_abi.js                             # MUST run after compile to update contracts/abi/
npx hardhat run scripts/deploy.js                      # Deploy locally
npx hardhat run scripts/deploy_testnet.js --network baseSepolia  # Deploy to Base Sepolia
```

### Python SDK Tests
```bash
pytest tests/                  # All tests
pytest tests/test_protocol.py  # Single file
```

### Experiments
```bash
python experiments/bench_latency.py                      # Off-chain latency
python experiments/bench_latency.py --onchain             # + on-chain (Hardhat)
python experiments/bench_latency.py --onchain --sepolia   # + Base Sepolia
python experiments/bench_gas.py                           # Gas costs (needs Hardhat node)
python experiments/bench_gas_sepolia.py                   # Gas costs on Sepolia
python experiments/trust_analysis.py                      # Trust comparison table
python experiments/scenario_demo.py                       # E2E 3-agent demo (needs .env with LLM key)
```

## Architecture

### Protocol State Machine (agent_sdk/protocol.py)
```
REGISTERED → DELEGATED → EXECUTED → SETTLED
                                    ↘ DISPUTED
```

| Stage | Contract | On-chain Operation |
|-------|----------|--------------------|
| REGISTER | AgentRegistry | Agent stakes ETH, declares capabilities |
| DELEGATE | PaymentChannel | Requester locks payment in escrow |
| EXECUTE | ResultAttestation | Provider attests result hash |
| SETTLE | PaymentChannel | Payment release or refund |
| DISPUTE | PaymentChannel.forceRelease | Auditor arbitrates |

### Key Components

- **contracts/** — Three Solidity contracts (Solidity 0.8.24). AgentRegistry handles identity/stake/reputation. PaymentChannel manages escrow with timeout and auditor dispute paths. ResultAttestation stores immutable result hashes with dispute flags.
- **contracts/abi/** — Exported JSON ABIs consumed by the Python SDK. Regenerated by `node scripts/export_abi.js`.
- **agent_sdk/** — Python SDK. `ChainClient` (chain.py) wraps Web3 contract calls. Three agent roles: `RequesterAgent`, `ProviderAgent`, `AuditorAgent`, all using a pluggable `LLMClient` (supports OpenAI, Anthropic, Ollama, MiniMax, Zhipu).
- **test/** — Hardhat/Mocha/Chai contract tests (JavaScript).
- **tests/** — pytest SDK tests (Python, mocked — no live LLM or blockchain needed).
- **experiments/** — Research benchmarks for the paper (latency, gas, trust analysis, scenario demo).
- **paper/** — IEEE-format LaTeX paper.
- **types/ethers-contracts/** — Auto-generated TypeScript types (do not edit).

### Critical Workflow
After modifying any Solidity contract: `compile` → `export_abi.js` → Python SDK picks up new ABIs. Skipping the ABI export means the SDK uses stale interfaces.

## Key Details

- `package.json` has `"type": "module"` — ES module syntax throughout JS.
- Contracts deployed on Base Sepolia; addresses recorded in `deployments.json`.
- `.env` (gitignored) holds secrets; `.env.example` is the template.
- The LLM provider is configured via `LLM_PROVIDER` env var (openai/anthropic/ollama/minimax/zhipu).
- `docs/superpowers/` contains historical design/plan docs.
