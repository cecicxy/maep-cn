实验 1：协议层有多快？

测量协议状态机本身的执行开销，对比纯 Python 基本操作

离链协议开销（微秒）

阶段 MAEP 耗时 基本操作耗时 开销倍数
委托 1.12 μs 0.23 μs 4.8x
执行 3.32 μs 1.08 μs 3.1x
结算 1.47 μs 0.22 μs 6.8x

这意味着什么：协议本身只增加约 1-3 微秒的开销（百万分之一秒），完全不影响实际使用。真正的瓶颈在大模型推理时间。

链上确认延迟（毫秒）

阶段 平均耗时 范围
注册 11.2 ms 5.1 – 28.0 ms
委托 7.8 ms 6.0 – 12.2 ms
执行 5.5 ms 4.6 – 6.1 ms
结算 6.0 ms 5.5 – 6.4 ms
合计 30.5 ms

这意味着什么：本地测试网总耗时约 30 毫秒，但在真实的 Base Sepolia 网络上，每个操作大约需要 2 秒（出块时间）。

实验 2：一次任务要花多少钱？

阶段 本地测试 Base Sepolia 差异
注册 138,625 138,625 0
委托 116,489 116,501 +12
执行 93,590 93,602 +12
结算 62,642 60,142 -2,500
合计 411,346 408,870 ~0.6%

这意味着什么：408,870 Gas × 0.006 gwei ≈ 0.00000245 ETH，按 ETH $3,000 计算约 $0.007/任务（不到 1 分钱）。注册费用可以分摊到多次任务中，实际每次任务成本更低。国内版完全不消耗 Gas。

实验 3：你需要信任谁？

每个协议在哪些环节需要信任第三方？信任的参与方越少，安全性越高。

环节 x402 Google A2A Fetch.ai MAEP
Agent 身份 HTTP 服务器 Google 平台 验证者集合 以太坊合约
支付 稳定币+服务器 无 FET+验证者 以太坊合约
结果验证 无 无 网络 以太坊合约
争议仲裁 无 平台 链上治理 可配置
需要信任几方 3 2 2 1

这意味着什么：MAEP 只需要信任以太坊网络本身（全球 90 万+ 验证者），不依赖任何公司或组织。国内版信任平台运营方——这是用去中心化换合规性的取舍。

实验 4：端到端跑一遍要多久？

场景 最终状态 耗时 说明
正常路径 已结算 1.84s 三个 Agent 顺利完成一次完整交易
争议路径 仲裁完成 4.20s 结果被拒绝，审计 Agent 介入仲裁

这意味着什么：绝大部分时间花在大模型推理上（生成结果和验证结果），协议本身的开销几乎可以忽略。

国内版：数据库操作有多快？

操作 平均耗时 中位数
创建任务（锁定余额） 21 μs 20 μs
提交结果 29 μs 28 μs

这意味着什么：所有操作都在 30 微秒内完成（千分之三十毫秒），比链上操作快约 1000 倍。这就是去掉区块链之后的速度优势。
