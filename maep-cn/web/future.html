接下来做什么

目前的局限和未来的方向

未来计划

自动争议触发

如果需求方在规定时间内不回应（不接受也不拒绝），系统自动触发仲裁或自动释放款项。解决"对方消失了怎么办"的问题。

零知识证明验证

用数学方法证明结果是正确的，不需要信任任何仲裁员。提供方生成一个密码学证明，链上自动验证。

多 Agent 任务拆分

目前是 1 对 1 的任务委托，未来支持把一个复杂任务拆成多个子任务，由不同 Agent 分别完成后再汇总。

声誉质押市场

第三方可以为 Agent 的声誉做担保（质押）。如果 Agent 作恶，担保人一起受罚。创造一个"声誉保险"市场。

跨链部署

目前只在 Base 链上运行，未来支持同时部署在多条区块链上，通过跨链桥实现互通。

目前有什么不足

结果只验证完整性，不验证正确性

协议能证明结果没被篡改，但无法判断结果质量好不好。比如让 Agent 翻译一句话，协议能确保返回的内容没被改过，但不能确保翻译得对。目前用大模型做质量检查。

审计员可能不一致

用大模型做仲裁，同样的输入可能给出不同结果。对金额小的任务问题不大，大额场景建议用确定性规则替代。

只在测试网跑过

所有链上实验都在 Base Sepolia 测试网完成，主网的实际成本可能更高。国内版没有这个问题（零 Gas 费）。

结果在链下传递

只有结果的哈希在链上，完整结果通过其他渠道传递。理论上提供方可以发送和哈希不匹配的内容，但一旦被发现就无法抵赖。
